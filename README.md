# JavaScript Learning Repository

## Overview
This repository contains a collection of hands-on JavaScript projects designed to build strong fundamentals in DOM manipulation, asynchronous programming, state management, and application architecture.

The tasks progress from basic interactivity to building full-featured applications, simulating real-world frontend development scenarios.

---

## Objectives

- Understand core JavaScript concepts through practical implementation
- Build interactive user interfaces using DOM manipulation
- Work with asynchronous operations using Fetch API and timers
- Manage application state effectively
- Implement modular and scalable code using ES6 modules
- Develop problem-solving skills for real-world frontend challenges

---

## Topics Covered

### Fundamentals
- DOM selection and manipulation
- Event handling and user interactions
- Form handling and validation

### UI Interactions
- To-do list with dynamic updates
- Calculator with expression handling
- Image gallery with modal/lightbox
- Drag and drop functionality using HTML5 API

### Asynchronous JavaScript
- Fetch API for data retrieval
- Handling JSON responses
- Error handling in async operations
- Simulating real-time behavior using `setTimeout`

### Application Development
- Weather application with API integration
- Quiz application with external JSON data
- Real-time chat simulation
- Infinite scrolling content loader

### Advanced Concepts
- Hash-based routing (SPA simulation)
- State management using JavaScript objects/arrays
- LocalStorage for persistence
- Modular JavaScript using ES6 modules

### Mini Project
- Full-featured eCommerce shopping cart with:
  - Product listing
  - Cart management
  - Dynamic pricing
  - Search/filter functionality
  - Persistent state

---

## Repository Structure

Each task is organized as:

javascript/task-x/

Each task includes:

- `index.html`
- `styles.css`
- `script.js`
- Additional files (e.g., JSON, modules) where required
- `README.md` explaining the task

---

## How to Run

1. Navigate to any task folder
2. Open `index.html` in a browser
3. For module-based tasks, ensure files are served via a local server if required

---

## Key Learnings

- Handling asynchronous workflows
